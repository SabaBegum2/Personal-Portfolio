Driveway Sealing Mangament System

This project involves designing a database and system for managing driveway-sealing services for contractor David Smith. Clients register with their personal details and receive a unique ID. They can submit requests for quotes with details such as property address, driveway size, proposed price, and images. David can respond with a counteroffer or rejection. Both parties can negotiate until an agreement is reached or the process is terminated. Accepted quotes create work orders, forming a contract. Once the work is completed, David generates a bill, which the client can pay or dispute. Bill negotiations can continue until resolved.

    • E-R Diagram: I designed an E-R diagram for the system, clearly outlining the entities (`Clients`, `Quotes`, `Orders`, and `Bills`), relationships, and constraints using arrows and thick lines for better representation.
    • Relational Model: I translated the E-R diagram into a relational model using `CREATE TABLE` statements, specifying primary keys, foreign keys, and constraints wherever applicable.
    • David Smith’s Dashboard: I implemented a dashboard for David Smith to:
      • View and manage quotes, orders, and bills.
      • Respond to the most recent quotes and bills.
      • Generate revenue reports for a specific period.
    • Client Dashboard: I implemented a client dashboard that allows clients to:
      • View their quotes, orders, and bills.
      • Respond to the most recent response from David Smith.
    • Query Implementations: I implemented the following queries:
      • Big Clients: Listed clients with the highest number of completed orders, showing results and explanations for changes in data.
      • Difficult Clients: Listed clients who submitted three requests but never followed up.
      • This Month Quotes: Listed all agreed quotes for December 2024.
      • Prospective Clients: Listed clients who registered but never submitted a request.
      • Largest Driveway: Listed locations of the largest driveways worked on.
      • Overdue Bills: Listed bills unpaid after one week of generation.
      • Bad Clients: Listed clients who never paid overdue bills.
      • Good Clients: Listed clients who paid bills within 24 hours of generation.
    • Testing and Verification: I tested each query, verified the results, and modified the database to demonstrate changes in the results, explaining the before-and-after scenarios.

    • Project Overview: Design a web-based system to manage driveway-sealing services for contractor David Smith. Clients register, request quotes, negotiate, and pay bills, while David manages quotes, orders, and billing via a dashboard.
    • Required Functionalities:
      • Dashboard for David Smith:
        • Check all incoming quote requests, their content, responses, and status.
        • View all orders of work, their content, and status.
        • Manage bills, including content and status.
        • Respond to the most recent quotes and bills.
        • Generate revenue reports for specified periods.
      • Dashboard for Clients:
        • View their quotes, orders, and bills.
        • Respond to the most recent response from David Smith.
    • Implementation Requirements:
      • Design an E-R diagram using arrows or thick lines to represent constraints, with clear assumptions and justifications.
      • Translate the E-R diagram into a relational model using `CREATE TABLE` statements, specifying primary keys, foreign keys, and constraints.
      • Develop a web-based interface for all functionalities described above. SQL execution via tools like MySQL Workbench is for debugging only.
    • Query Functionality:
      • Big Clients: List the clients with the highest number of completed orders. In case of a tie, list all top clients.
      • Difficult Clients: List clients who submitted three requests but never followed up.
      • This Month Quotes: List all agreed quotes for the current month (e.g., December 2024).
      • Prospective Clients: List clients who registered but never submitted a quote request.
      • Largest Driveway: List locations of the largest driveways David Smith worked on. If tied, list all locations.
      • Overdue Bills: List bills unpaid after one week of generation.
      • Bad Clients: List clients who never paid overdue bills. Exclude clients with no bills.
      • Good Clients: List clients who paid bills within 24 hours of generation.
    • Additional Requirements:
      • Include simple GUI interfaces for all functionalities.
      • All interactions must occur via the web-based interface, not through direct SQL execution.
